## How Explanations Are Documented

For each variance requiring explanation:
1. **State the amount** of the variance
2. **State the primary driver** (volume, pricing, timing, one-off, new customer/contract, cost change)
3. **State whether it is recurring or one-off**
4. **Reference supporting evidence** (invoice, contract, payroll, correspondence)

Format: `[GL range] [Category]: [Amount] variance driven by [driver]. [Recurring/one-off]. Evidence: [reference].`

---

## How Conclusions Are Recorded

Each section ends with one of:
- **Explained** — all variances within threshold or explained above threshold
- **Partially explained** — some material variances remain without explanation → list in exception log
- **Not reviewed** — section not completed → blocks close

---

## How Exceptions Are Carried Forward

Unexplained variances are recorded in the Review Sign-Off and Exception Log (`templates/review-signoff-and-exception-log-template.md`) with:
- Exception description
- Amount
- Owner
- Required action
- Deadline
- Whether it blocks close
